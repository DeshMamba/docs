Actions are the steps your workflow performs after the trigger fires. Chain as many actions as needed and use connection conditions to branch between paths.

## Communication

<AccordionGroup>

  <Accordion title="Call Lead">
    Places an AI-powered outbound call to the contact.

    **Configuration:**
    - **Assistant** — the agent that handles the call
    - **Phone number** — the number to dial (defaults to `{phone_number}`)
    - **Recipient name** — name passed to the agent (defaults to `{name}`)

    **Outputs:** Call Completed, Call Failed, Voicemail Detected. Connect different nodes to each branch to handle each outcome.

    **Use case:** Call a new Facebook lead within seconds of form submission, or call back a contact who requested a follow-up.
  </Accordion>

  <Accordion title="Send SMS">
    Sends an SMS to the contact using your connected phone number.

    **Configuration:**
    - **To** — recipient phone number (defaults to `{phone_number}`)
    - **Message body** — the text to send. Use the variable picker to insert dynamic values like `{name}`, `{summary}`, or `{appointment.start_time}`.

    <Tip>
      Keep messages under 160 characters to avoid carrier splitting. Variable values count toward the limit at their resolved length.
    </Tip>
  </Accordion>

  <Accordion title="Slack">
    Sends a message to a Slack channel or user. Requires Slack to be connected under **Settings → Integrations**.

    **Configuration:**
    - **Channel** — channel name (e.g., `#general`) or user ID
    - **Message** — message content. Supports variables.
  </Accordion>

</AccordionGroup>

## Logic & flow

<AccordionGroup>

  <Accordion title="Condition">
    Evaluates a true/false condition and routes the workflow down one of two branches.

    **Configuration:**
    - **Condition expression** — the rule to evaluate using variable syntax (e.g., `{sentiment} == 'positive'`)

    The node has two output handles: **True** and **False**. Connect different nodes to each to define what happens in each case.
  </Accordion>

  <Accordion title="Router">
    Splits the workflow into multiple branches based on per-branch conditions. Use Router when you need more than two paths.

    **Configuration:**
    - Add a branch for each path
    - Set a condition expression for each branch: variable, operator (equals, not_equals, contains, not_contains), and value
    - Connect each branch to its next action

    **Use case:** Route contacts to different follow-up sequences based on call outcome — one path for booked, one for interested, one for not interested.
  </Accordion>

  <Accordion title="Wait">
    Pauses workflow execution for a set duration before continuing to the next node.

    **Configuration:**
    - **Duration** — how long to wait
    - **Unit** — seconds, minutes, hours, or days

    **Use case:** Wait 30 minutes after a call before sending a follow-up SMS.
  </Accordion>

  <Accordion title="Tag Contact">
    Applies a tag to the contact in your CRM.

    **Configuration:**
    - **Tag** — the label to apply

    **Use case:** Tag a contact as "interested" when the call outcome matches, then use that tag to filter a follow-up campaign list.
  </Accordion>

</AccordionGroup>

## CRM

<AccordionGroup>

  <Accordion title="HubSpot">
    Writes data to your connected HubSpot account. Requires HubSpot to be connected under **Settings → Integrations**.

    **Available actions:**
    - **Create Contact** — creates a new HubSpot contact with fields mapped from variables
    - **Update Contact** — updates an existing contact's properties
    - **Create Deal** — opens a new deal and associates it with a contact
    - **Create Company** — creates a new company record
    - **Create Associations** — links two HubSpot objects (e.g., a contact to a deal)
  </Accordion>

  <Accordion title="GoHighLevel">
    Writes data to your connected GoHighLevel account. Requires GoHighLevel to be connected under **Settings → Integrations**.

    **Available actions:**
    - **Create Contact** — creates a new contact
    - **Update Contact** — updates an existing contact's fields
    - **Add to Campaign** — enrolls the contact in a GoHighLevel campaign
    - **Add Tag** — applies a tag to the contact in GoHighLevel
  </Accordion>

</AccordionGroup>

## Connections

<AccordionGroup>

  <Accordion title="HTTP Request">
    Makes an HTTP call to any external API or webhook URL. Use this to integrate with tools not natively supported in the Composer.

    **Configuration:**
    - **URL** — the endpoint to call. Supports variables (e.g., `https://api.example.com/contacts/{phone_number}`)
    - **Method** — GET, POST, PUT, PATCH, or DELETE
    - **Headers** — key-value pairs for authorization or content type
    - **Body** — JSON payload for POST/PUT/PATCH requests. Supports variables.
    - **Expected outputs** — response field names to extract as variables for downstream nodes

    **Use case:** Push call summaries to a custom webhook, update an external database, or trigger a notification in a third-party tool.
  </Accordion>

</AccordionGroup>
