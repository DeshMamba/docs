**Actions** are steps that the workflow performs. This page documents all available node types and their configuration options.

<Tabs>
  <Tab title="Actions">
    Actions are the steps your workflow performs after the trigger fires. You can chain as many actions as needed, and use connection conditions to branch your logic.

    <AccordionGroup>
      <Accordion title="Call Lead">
        Initiates an AI-powered outbound call to the contact. The call is placed using your connected Twilio number and dispatched to the AI assistant you select.

        **Configuration:**

        - **Assistant** — the AI assistant that will handle the call
        - **Phone number** — the number to dial (defaults to `{phone_number}` to use the contact's number on file)
        - **Call script / context** — optional instructions or context to pass to the assistant for this specific call

        **Use case:** Automatically call a new Facebook lead within seconds of form submission, or call back a contact who requested a follow-up during a previous call.
      </Accordion>

      <Accordion title="Send SMS (Twilio)">
        Sends an SMS message to the contact using your connected Twilio account.

        **Configuration:**

        - **To** — the recipient's phone number (defaults to `{phone_number}`)
        - **Message body** — the text to send. Use the variable picker to insert dynamic values like `{name}`, `{summary}`, or `{appointment.start_time}`.

        **Requirements:** A Twilio account and phone number must be connected in **Settings → Integrations**.

        <Tip>
          Keep SMS messages under 160 characters to avoid carrier splitting. Variable values count toward the character limit at their resolved length.
        </Tip>
      </Accordion>

      <Accordion title="Condition">
        Evaluates a true/false condition and routes the workflow down one of two branches. Use this when you need to take different actions depending on a value — for example, only sending a confirmation SMS when `{outcome}` equals "booked".

        **Configuration:**

        - **Condition expression** — the rule to evaluate, using variable syntax (for example, `{sentiment} == 'positive'`)

        The node has two output handles: **True** and **False**. Connect different nodes to each handle to define what happens in each case.
      </Accordion>

      <Accordion title="Router">
        Splits the workflow into multiple branches based on per-branch conditions. Unlike the Condition node (which is binary), a Router supports as many branches as you need.

        **Configuration:**

        - Add a branch for each path you want to support
        - Write a condition expression for each branch
        - Connect each branch handle to the next action for that path

        **Use case:** Route a contact to different follow-up sequences based on call outcome — one path for "booked", one for "interested but not booked", and one for "not interested".
      </Accordion>

      <Accordion title="Wait">
        Pauses workflow execution for a specified duration before continuing to the next node.

        **Configuration:**

        - **Duration** — how long to wait
        - **Unit** — seconds, minutes, or hours

        **Use case:** Wait 30 minutes after a call before sending a follow-up SMS, so the message feels timely rather than immediate.

        <Note>
          The actual delay may be slightly longer than configured during periods of high platform load.
        </Note>
      </Accordion>

      <Accordion title="Tag Contact">
        Applies a tag to the current contact in your CRM. Tags help you segment contacts, filter lists, and trigger other automations.

        **Configuration:**

        - **Tag** — the label to apply to the contact

        **Use case:** Tag a contact as "interested" when the call outcome matches, then use that tag to filter your contact list for a follow-up campaign.
      </Accordion>

      <Accordion title="HubSpot CRM actions">
        Performs a write operation in your connected HubSpot account.

        **Available actions:**

        - **Create Contact** — creates a new HubSpot contact with fields mapped from variables
        - **Update Contact** — updates an existing contact's properties
        - **Create Deal** — creates a new deal and associates it with a contact
        - **Create Company** — creates a new company record
        - **Create Associations** — links two existing HubSpot objects (for example, a contact to a deal)

        **Requirements:** HubSpot must be connected in **Settings → Integrations** with a valid OAuth connection.
      </Accordion>

      <Accordion title="GoHighLevel CRM actions">
        Performs a write operation in your connected GoHighLevel account.

        **Available actions:**

        - **Create Contact** — creates a new contact
        - **Update Contact** — updates an existing contact's fields
        - **Add to Campaign** — enrolls the contact in a GoHighLevel campaign
        - **Add Tag** — applies a tag to the contact in GoHighLevel

        **Requirements:** GoHighLevel must be connected in **Settings → Integrations**.
      </Accordion>

      <Accordion title="HTTP Request">
        Makes an HTTP call to any external API or webhook URL. Use this to integrate with services that aren't natively supported in the Composer.

        **Available methods:**

        - GET, POST, PUT, PATCH, DELETE

        **Configuration:**

        - **URL** — the endpoint to call. Supports variables (for example, `https://api.example.com/contacts/{phone_number}`)
        - **Method** — the HTTP method to use
        - **Headers** — key-value pairs for authorization or content type
        - **Body** — JSON payload for POST/PUT/PATCH requests. Use variables to inject dynamic data.

        **Use case:** Push call summaries to a custom webhook, update a record in a database, or trigger a notification in a third-party tool.
      </Accordion>
    </AccordionGroup>
  </Tab>
</Tabs>
