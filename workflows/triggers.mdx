A trigger is the entry point of every workflow. It defines the event that starts the workflow. Each workflow has exactly one trigger.

<AccordionGroup>

  <Accordion title="End of call">
    Starts the workflow after a call handled by your agent ends. The most common trigger for post-call automation: sending follow-up SMS, updating your CRM, or placing a callback.

    **Configuration:** Select the agent this trigger watches. The workflow fires once per completed call.

    **Available variables:** All Call Data variables and Contact Info variables. See [Variables](/workflows/variables).
  </Accordion>

  <Accordion title="Campaign">
    Starts the workflow when a contact enters a campaign. Use this to build sequences that run alongside outbound dialing — for example, sending an SMS before a call attempt or tagging contacts based on outcome.

    **Configuration:** Select the campaign this trigger is associated with. The workflow fires automatically as the campaign processes each contact.

    <Note>
      Campaign-triggered workflows are linked to a campaign, not to a specific agent. They run in parallel with the campaign's dialing logic.
    </Note>
  </Accordion>

  <Accordion title="Manual">
    Starts the workflow by hand from the dashboard. Use this for testing your workflow or for one-off operations that don't need to be automated.

    **Configuration:** No setup required. Open the workflow in the Composer and click **Run manually**.
  </Accordion>

  <Accordion title="HubSpot">
    Starts the workflow when a specific event occurs in your connected HubSpot account. Requires HubSpot to be connected under **Settings → Integrations**.

    **Available events:**

    Contacts: New Contact, Contact Recently Created or Updated, New Contact in List, New Contact Property Change

    Companies: New Company, Company Recently Created or Updated, New Company Property Change

    Deals: New Deal, Updated Deal Stage, New Deal Property Change

    Line items & products: New Line Item, Line Item Recently Created or Updated, New Product, Product Recently Created or Updated

    Tickets: New Ticket, New Ticket Property Change

    Tasks & engagements: New Task, New Engagement

    Email: New Email Event, New Email Subscriptions Timeline

    Forms & content: New Form Submission, New Blog Article

    Custom objects: New Custom Object, New Custom Object Property Change

    **Available variables:** HubSpot Contact variables. See [Variables](/workflows/variables).
  </Accordion>

  <Accordion title="GoHighLevel">
    Starts the workflow when a contact is created in your connected GoHighLevel account. Requires GoHighLevel to be connected under **Settings → Integrations**.

    **Available event:** Contact Created.

    **Available variables:** Contact details passed through from GoHighLevel.
  </Accordion>

  <Accordion title="Facebook Lead">
    Starts the workflow when a new lead is captured from a Facebook Lead Ad. Requires Facebook to be connected under **Settings → Integrations**.

    **Configuration:** Select the Facebook page and lead form to watch.

    **Available variables:** Facebook Lead variables including name, email, phone, ad name, and campaign name. See [Variables](/workflows/variables).
  </Accordion>

  <Accordion title="Schedule">
    Starts the workflow at a specific scheduled time. Use this for recurring automations that are not tied to a call or contact event.

    **Configuration:** Set the date, time, and recurrence for the trigger.
  </Accordion>

</AccordionGroup>
