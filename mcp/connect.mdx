---
title: "Connect an AI Client to Chattrvox MCP Server"
description: "Step-by-step guide to connecting Claude Desktop, Cursor, or Windsurf to the Chattrvox MCP endpoint using an API key in about two minutes."
---

Connecting your AI client to Chattrvox takes about two minutes. You generate an API key in your account settings, then paste a short configuration snippet into your client. Once connected, your AI assistant can create workflows, search knowledge bases, and manage your account through natural language.

<Steps>
  <Step title="Generate an API key">
    In Chattrvox, go to **Settings → Integrations → API Keys** and click **Generate New Key**.

    Give the key a label that identifies your client — for example, `Claude Desktop` or `Cursor`. Click **Create**, then copy the key from the highlighted field.

    <Warning>
      Your API key is displayed only once. Copy it before closing the dialog — there is no way to retrieve the raw key afterward. If you lose it, revoke the key and generate a new one.
    </Warning>

    Your key follows the format `wr_live_` followed by 32 random characters. In the API Keys list, only the first twelve characters are shown (for example, `wr_live_a8f3...`).
  </Step>

  <Step title="Note your MCP server URL">
    The Chattrvox MCP server is hosted at a permanent URL — you do not need to run anything locally:

    ```
    https://mcp.waverunner.digital/mcp
    ```

    Copy this URL. You will paste it into your client in the next step.
  </Step>

  <Step title="Add the server to your AI client">
    Configuration differs slightly between clients. Follow the tab for the client you are setting up.

    <Tabs>
      <Tab title="Claude Desktop">
        Claude Desktop reads its MCP configuration from a JSON file on your machine.

        **Find the config file:**

        | Platform | Path |
        |----------|------|
        | macOS | `~/Library/Application Support/Claude/claude_desktop_config.json` |
        | Windows | `%APPDATA%\Claude\claude_desktop_config.json` |

        Open the file in any text editor (create it if it does not exist). Add the `chattrvox` entry under `mcpServers`:

        ```json
        {
          "mcpServers": {
            "chattrvox": {
              "url": "https://mcp.waverunner.digital/mcp",
              "headers": {
                "Authorization": "Bearer wr_live_xxxxxxxxxxxxxxxxxxxx"
              }
            }
          }
        }
        ```

        Replace `wr_live_xxxxxxxxxxxxxxxxxxxx` with your actual API key.

        Save the file, then **quit and relaunch Claude Desktop**. The Chattrvox tools appear in Claude's tool list after restart.

        <Tip>
          If you already have other MCP servers in your config, add the `chattrvox` block alongside them inside the same `mcpServers` object.
        </Tip>
      </Tab>

      <Tab title="Cursor">
        Open **Cursor → Settings** (or press `Cmd+,` on macOS / `Ctrl+,` on Windows), then navigate to the **MCP** section.

        Click **Add Server** and fill in the fields:

        | Field | Value |
        |-------|-------|
        | **Name** | `chattrvox` |
        | **URL** | `https://mcp.waverunner.digital/mcp` |
        | **Auth header** | `Authorization: Bearer wr_live_xxxxxxxxxxxxxxxxxxxx` |

        Replace `wr_live_xxxxxxxxxxxxxxxxxxxx` with your API key. Save and close settings. Cursor connects immediately — no restart required.
      </Tab>

      <Tab title="Windsurf">
        Open **Windsurf → Preferences** and navigate to the **MCP Servers** section.

        Click **Add MCP Server** and enter:

        | Field | Value |
        |-------|-------|
        | **Name** | `chattrvox` |
        | **URL** | `https://mcp.waverunner.digital/mcp` |
        | **Authorization** | `Bearer wr_live_xxxxxxxxxxxxxxxxxxxx` |

        Replace `wr_live_xxxxxxxxxxxxxxxxxxxx` with your API key. Save the configuration. Windsurf registers the server and makes the tools available in the next chat session.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Test the connection">
    With your client open, type a simple prompt to confirm the tools are reachable:

    > "List my Chattrvox workflows."

    Your AI assistant should call the `list_workflows` tool and return a list of your workflows (or an empty list if you have not created any yet). This confirms the server URL and API key are both correct.

    You can also verify the connection programmatically by calling the health check endpoint directly:

    ```bash
    curl https://mcp.waverunner.digital/mcp/health \
      -H "Authorization: Bearer wr_live_xxxxxxxxxxxxxxxxxxxx"
    ```

    A successful response looks like:

    ```json
    {
      "status": "ok",
      "user": "you@example.com",
      "workspace": "your-workspace-name",
      "tools": 9
    }
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="My AI client says the tools are not available">
    Check that the server URL is exactly `https://mcp.waverunner.digital/mcp` — no trailing slash, no extra path segments. Then confirm the `Authorization` header uses the `Bearer` prefix followed by your full API key. In Claude Desktop, restart the app after saving `claude_desktop_config.json`.
  </Accordion>

  <Accordion title="I get an authentication error">
    Your API key may have been revoked or entered incorrectly. Go to **Settings → Integrations → API Keys** in Chattrvox and check that the key you are using is still listed and shows no revocation date. If it was revoked, generate a new key and update your client configuration.
  </Accordion>

  <Accordion title="The health check returns a network error">
    Confirm that your machine can reach `mcp.waverunner.digital` over HTTPS. If you are behind a corporate firewall or VPN, the domain may need to be allow-listed. Contact your network administrator if outbound HTTPS to that domain is blocked.
  </Accordion>

  <Accordion title="I copied the wrong URL">
    The correct server URL is always `https://mcp.waverunner.digital/mcp`. The Chattrvox dashboard also shows this URL in **Settings → Integrations → MCP Connection** for easy copying.
  </Accordion>
</AccordionGroup>
