The HubSpot integration links your HubSpot portal to Chattrvox through OAuth. Once connected, you can fire automation workflows from over 20 HubSpot event types — contact created, deal stage changed, form submitted, and more — and write call summaries, transcripts, and outcomes back to the corresponding HubSpot contact record automatically.

## Connecting HubSpot

<Steps>
  <Step title="Open the integrations page">
    In Chattrvox, click **Settings** in the navbar, then select the **Integrations** tab.
  </Step>
  <Step title="Click Connect on the HubSpot card">
    Find the HubSpot card in the **CRM** section and click **Connect**. Chattrvox redirects you to HubSpot's OAuth authorization screen.
  </Step>
  <Step title="Authorize Chattrvox in HubSpot">
    Log in to your HubSpot account if prompted, then review the requested permissions and click **Grant access**. Chattrvox requests read and write access to contacts, companies, deals, leads, line items, products, appointments, lists, tickets, conversations, and OAuth.
  </Step>
  <Step title="Confirm the connection">
    HubSpot redirects you back to Chattrvox. The HubSpot card status changes to **Connected**, showing your portal ID.
  </Step>
</Steps>

<Note>
  You must have **Super Admin** or **CRM Admin** permissions in HubSpot to authorize an OAuth app for your portal.
</Note>

## What the integration enables

### HubSpot workflow triggers

After connecting, you can use HubSpot events as the starting point for any Chattrvox workflow. In the **Workflow Builder**, add a **Trigger** node and select **HubSpot** from the integrations list.

Available HubSpot trigger events:

| Category | Trigger |
| --- | --- |
| Contacts | New Contact |
| Contacts | Contact Recently Created or Updated |
| Contacts | New Contact in List |
| Contacts | New Contact Property Change |
| Companies | New Company |
| Companies | Company Recently Created or Updated |
| Companies | New Company Property Change |
| Deals | New Deal |
| Deals | Updated Deal Stage |
| Deals | New Deal Property Change |
| Line items | New Line Item |
| Line items | Line Item Recently Created or Updated |
| Products | New Product |
| Products | Product Recently Created or Updated |
| Tickets | New Ticket |
| Tickets | New Ticket Property Change |
| Tasks | New Task |
| Engagements | New Engagement |
| Email | New Email Event |
| Email | New Email Subscriptions Timeline |
| Forms | New Form Submission |
| Blog | New COS Blog Article |
| Custom objects | New Custom Object |
| Custom objects | New Custom Object Property Change |

When a matching event fires in HubSpot, Chattrvox executes any active workflow that uses that trigger.

### HubSpot CRM action nodes

Inside a workflow, you can add **HubSpot** action nodes to write data back to your CRM:

- **Create Contact** — creates a new contact in HubSpot with the data available from the workflow context
- **Update Contact** — updates properties on an existing contact record
- **Create Company** — creates a company in HubSpot
- **Create Deal** — opens a new deal and sets the initial stage
- **Create Associations** — links objects together (for example, associating a contact with a company)

### Call outcome sync

When a workflow is triggered by a completed call (via the **End of Call** trigger), the following call data is available to pass to HubSpot action nodes:

- **Summary** — the AI-generated call summary
- **Transcript** — the full call transcript
- **Outcome** — the call result (e.g., booked, not interested, callback requested)
- **Sentiment** — detected caller sentiment
- **Duration** — call length in seconds

Use these in a **Create Note** or **Update Contact** node to keep HubSpot records current without any manual effort.

## Using HubSpot contact variables

When a workflow starts from a HubSpot contact event, Chattrvox makes the contact's properties available as variables you can insert into any text field — SMS message body, call script, email content, or webhook payload.

To insert a variable, click **\+ Insert Variable** inside any text field in the **Node Config Panel** and select from the **HubSpot Contact** category.

Available HubSpot contact variables:

| Variable | Description |
| --- | --- |
| `{firstname}` | Contact's first name |
| `{lastname}` | Contact's last name |
| `{email}` | Contact's email address |
| `{phone}` | Contact's phone number |
| `{company}` | Associated company name |
| `{lifecyclestage}` | HubSpot lifecycle stage |
| `{jobtitle}` | Job title |
| `{city}` | City |
| `{state}` | State or region |
| `{zip}` | Postal code |
| `{country}` | Country |

<Tip>
  You can combine HubSpot variables with call data variables in the same text field. For example, an SMS message body could read: "Hi `{firstname}`, here's a summary of our call: `{summary}`."
</Tip>

## Two-way contact sync

When Chattrvox receives a HubSpot contact event, it automatically checks whether the contact already exists in your Chattrvox contact list by matching phone number variants. If a match is found, Chattrvox updates the contact record. If no match is found, Chattrvox creates the contact and adds it to a **CRM Contacts** list. HubSpot is treated as the source of truth when the integration is active.

## Disconnecting HubSpot

<Steps>
  <Step title="Open the integrations page">
    Go to **Settings → Integrations**.
  </Step>
  <Step title="Click Configure on the HubSpot card">
    The card shows **Configure** when a connection is active.
  </Step>
  <Step title="Remove the connection">
    Click **Disconnect** or the trash icon next to the connection entry. Chattrvox immediately stops receiving webhook events from that HubSpot portal.
  </Step>
</Steps>

<Warning>
  Disconnecting HubSpot does not pause active workflows. Workflows that use a HubSpot trigger will no longer fire, but they remain in draft status. Re-connecting HubSpot and re-deploying the workflow restores full functionality.
</Warning>
