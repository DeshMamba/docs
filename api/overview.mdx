The Chattrvox REST API gives you programmatic access to the same capabilities available in the dashboard: create and trigger workflows, manage contacts, search knowledge bases, control campaigns, and query usage minutes. All endpoints are served under a single base URL and follow consistent conventions for authentication, headers, and error responses.

## Base URL

```
https://your-domain.com/api/v1
```

Replace `your-domain.com` with your Chattrvox domain or custom white-label domain. All endpoints are relative to this base.

## Authentication

Every request requires an `Authorization` header with a Bearer token. Chattrvox supports two token types:

- **API keys** (`wr_live_...`) — recommended for integrations, scripts, and MCP clients
- **JWT tokens** — session tokens for browser-based requests

See [Authentication](/api/authentication) for details on obtaining and using each token type.

```bash
curl https://your-domain.com/api/v1/workflows \
  -H "Authorization: Bearer wr_live_xxxx"
```

## Workspace scoping

Chattrvox is multi-workspace. When you operate within a specific workspace, pass its UUID in the `x-workspace-id` header. Omitting this header targets your default (personal) context.

```bash
curl https://your-domain.com/api/v1/workflows \
  -H "Authorization: Bearer wr_live_xxxx" \
  -H "x-workspace-id: 9f4e2a1b-3c7d-4f8e-a2b1-0c9d8e7f6a5b"
```

## Rate limits

The API enforces rate limits per API key. If you exceed the limit, you receive a `429 Too Many Requests` response. Implement exponential backoff and retry logic in your integration.

| Status | Meaning |
|--------|---------|
| `429`  | Rate limit exceeded — wait before retrying |

## Error responses

All errors follow the same structure. Check the HTTP status code first, then read the `error` field for a human-readable message.

```json
{
  "error": "Workflow not found in this workspace"
}
```

Some endpoints use `success: false` with a `message` field:

```json
{
  "success": false,
  "message": "Name is required"
}
```

### Common status codes

| Code | Meaning |
|------|---------|
| `200` | Request succeeded |
| `400` | Bad request — missing or invalid parameters |
| `401` | Unauthorized — missing or invalid token |
| `403` | Forbidden — valid token but insufficient permissions |
| `404` | Resource not found |
| `429` | Rate limit exceeded |
| `500` | Server error |
